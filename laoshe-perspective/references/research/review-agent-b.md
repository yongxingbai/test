# 评审报告：老舍写作心智模型 SKILL.md

## 1. 激活触发条件

**现有 triggers（8个）：** 老舍、老舍的写作方法、京味写作、口语化写作、老舍式叙事、含泪的笑、如何写好对话、文学语言锤炼

**分析：** triggers 覆盖了核心使用场景，但有以下缺口：

- **缺"创作帮助类"触发词：** 用户不会直接说"老舍"，但可能在写作中遇到具体问题。缺了：`写作卡壳`、`人物写不活`、`对话写不好`、`语言太生硬`
- **缺"作品分析类"触发词：** 用户讨论老舍作品时也需要激活。缺了：`骆驼祥子`、`茶馆`、`四世同堂`、`正红旗下`、`老舍作品分析`
- **缺"文学批评类"触发词：** 缺了：`京味文学`、`现代文学`、`北京话写作`、`白话文写作`

**建议在 triggers 中补充的条目：**
```
- 写作卡壳
- 人物塑造
- 对话怎么写
- 骆驼祥子
- 茶馆
- 京味文学
```

## 2. 角色扮演规则的可操作性

### 问题路由机制：有但薄弱

回答工作流（Agentic Protocol）定义了问题分类（需要事实/方法论/混合），这是好的。但**缺少更细粒度的路由**：

- **缺少"用户问的是老舍生平/作品信息"时的路由：** 目前只有写作方法论有清晰路径。如果用户问"老舍和鲁迅关系如何"，走哪个模型？没有说明。
- **缺少"用户求助的不是写作，而是生活/职场/管理"时的路由：** 用户可能问"老舍先生，我该不该辞职写作？"——这属于人生建议，不是写作方法论。skill 没有定义这种情况如何处理。
- **缺少"用户的问题触及老舍知识盲区"时的路由：** 用户问"你怎么看海明威的冰山理论？"——老舍应该知道（他读过很多外国文学），但skill没定义"触及智识边界时怎么做"。

### 频率约束：严重缺失

角色扮演规则第3条说"带幽默感"，心智模型2说"幽默是调味不是主食"，但**缺少具体的频率约束规则**：

- 什么样的写作问题**不该**用幽默？用户讨论沉重的社会议题（自杀、战争、贫困）时，老舍的"含泪的笑"合适吗？什么时候幽默会变成冒犯？
- 什么时候不该用口语化？用户问的如果是学术写作、技术写作，口语化风格可能不适用。
- **缺少"感知用户情绪"的约束：** 如果用户的问题本身很沉重（如"我父亲去世了，想写悼词"），老舍应该切换模式。

### 失败预防：部分覆盖但有缺口

- 角色扮演规则第4条"遇到不知道的事就直说不知道"——好。
- 诚实边界第6条"老舍不是所有人都喜欢的"——好。
- **但缺少"当用户的问题涉及虚构/幻想/科幻写作"时的处理：** 心智模型3的局限性中提到了"不适用于科幻、奇幻、历史小说"，但如果用户问的就是这类问题，skill 没有说应该拒绝回答还是调整方式。
- **缺少"当用户的问题超出1966年知识范围"的具体兜底策略：** 用户问"你怎么看AI写作？"——规则说"不知道，没赶上"。但更优雅的做法是：基于你熟悉的写作原则给出你的推测，同时承认你不知道AI。

## 3. 缺失的关键信息

**最大的缺失：缺少"写作工作流"或"具体写作流程"**

skill 详细介绍了老舍的6个心智模型和10条启发式，都是"原则"。但老舍**实际上是怎么写一篇小说的？** 从构思到修改的完整流程是什么？这是用户最需要的——拿到就能用的步骤。目前缺少：

- **老舍的写作流程：** 从素材收集 → 构思提纲 → 写初稿 → 放一放 → 修改 → 朗读修改 → 定稿。这个流程没有明确呈现。
- **老舍的修改习惯：** 老舍反复强调修改的重要，但skill只在规则1（朗读一遍）中轻轻带过。老舍的修改具体怎么做的？他改什么？
- **缺少"写作诊断"功能：** 用户给一段文字，老舍应该怎么诊断问题？没有明确的诊断框架。

**其他缺失：**

- **缺少"文体切换指南"：** 老舍写散文、小说、话剧、通俗文艺的方法各不相同。skill 没有区分文体。
- **缺少"和当代写作场景的映射"：** 诚实边界第2条提到"有时代局限"，但具体怎么映射？比如"公众号文章怎么应用老舍的方法"——如果用户问，skill 没有指导。
- **"含泪的笑"没有量化标准：** 什么时候笑的比例太高？什么时候泪的比例太高？没有可操作的判断标准。

## 4. 使用体验建议

**好的方面：**
- 身份卡写得生动，有代入感
- 心智模型的结构好（一句话定义+来源证据+应用方式+局限性）
- 反模式和内在张力是有深度的
- 诚实边界很有价值

**让人困惑的地方：**

- **"回答工作流"和"角色扮演规则"的关系不清晰：** Step 1-3 定义了一个回答流程，但角色扮演规则第3条说"用老舍的语气说话"。当用户问事实性问题时，是先做研究再回答，还是先用老舍的语气聊？步骤之间没有优先级说明。
- **"表达DNA"的定位模糊：** 它是给AI用的（模仿老舍语气），还是给用户用的（理解老舍风格）？从位置看像是前者，但没有任何说明。用户可能困惑：这一节是让我读的，还是让AI读的？
- **"时间线"的用途不明确：** 是帮助AI理解老舍生平，还是给用户参考？如果是给AI的，应该放在"研究维度"附近；如果是给用户的，现在的放置位置（心智模型和价值观之间）有些突兀。

**需要加的使用说明/注意事项：**

1. **告诉用户什么时候该用、什么时候不该用这个skill：** 适合叙事写作、文学批评、人物塑造；不适合学术写作、技术文档、正式公文。
2. **告诉用户如何和这个skill互动效果最好：** 提供具体的写作片段 + 具体问题，比笼统地问"教我写小说"效果好。
3. **说明"表达DNA"的用途：** 如果用户想模仿老舍风格写一段文字，可以怎么利用这一节。

### 改动2：在角色扮演规则后增加"使用边界约束"

**位置：** 第33行之后（角色扮演规则之后，回答工作流之前）

**新增内容：**

```markdown
### 使用边界约束

1. **感知用户情绪**：如果用户的问题涉及丧亲、疾病、战争等沉重议题，减少幽默比例。此时"含泪的笑"不是"笑"，是"含泪"优先。老舍的幽默是调味，不是所有场合都需要。

2. **文体识别**：如果用户问的是学术写作、技术文档、正式公文，主动说明"我的方法主要适用于叙事性文学，用于你的场景需要调整"——不要强行套用口语化写作。

3. **超出知识边界的处理**：遇到1966年之后的事件或你完全没读过的话题，先说明"我没赶上/没读过"，然后基于你熟悉的写作原则给出推测——"不过照我的经验，大概是这样……"。不编造，但不沉默。

4. **非写作类问题**：如果用户问的是人生选择、职场困惑（"该不该辞职写作"），先作为"写家"给出建议，然后转向写作方法论——因为这才是你的专长。
```

### 改动3：在回答工作流中增加"Step 0: 用户意图判断"并新增"写作流程"附录

**位置：** Step 1 之前，以及在"决策启发式"之后

**Step 0 新增内容（放在 Step 1 之前）：**

```markdown
### Step 0: 用户意图判断

在回答之前，先判断用户的核心意图：

| 意图类型 | 特征 | 响应策略 |
|---------|------|---------|
| **求助写作** | 用户给了一段文字或描述了一个写作困难 | → 用心智模型诊断 → 给出具体修改建议 |
| **请教方法** | 用户问"怎么写对话""怎么让语言生动" | → 直接跳到Step 3，用心智模型回答 |
| **了解老舍** | 用户问老舍生平、作品、与其他人物的关系 | → 用身份卡和时间线回答 → 转向写作方法论（"你问这个，是想在写作中用上？"） |
| **探讨文学** | 用户讨论老舍与其他作家的比较、文学流派 | → 用智识谱系 + 温和批判模型回答 |
| **求助非写作** | 用户问人生建议、职场问题 | → 先用"写家"视角回答 → 引导到写作 |
| **评价作品** | 用户请老舍评价一段文字或一篇小说 | → 用心智模型逐条诊断 → 给出改进方向 |
```

**"老舍的写作流程"新增内容（放在"决策启发式"之后、表达DNA之前）：**

```markdown
### 附录：老舍的写作流程（实用指南）

这是老舍自己从构思到定稿的实际工作流程：

1. **素材收集**：在生活中观察、记录。听到一句有意思的话、看到一个有趣的人，就记下来。不记就忘了。
2. **构思提纲**：想清楚了再动笔。老舍写《骆驼祥子》前想了一年多，把故事从头到尾想透了。
3. **写初稿**：一口气写下去，不停下来修改。初稿的任务是把想法变成文字，不是写出精品。
4. **放一放**：写完后放几天。老舍说"越放越冷"，冷下来才能看出问题。
5. **朗读修改**：逐句朗读，不顺口的就改。这是老舍最核心的修改方法。
6. **再放再改**：好文章是改出来的。老舍的《骆驼祥子》改了好几遍。
7. **定稿**：改到自己觉得"不能再改了"为止。
```
